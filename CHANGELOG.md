# Changelog

ANIMA Zero 版本记录。**保持简洁——每版只说重点;具体改了什么,查 git commit。** 格式参考 [Keep a Changelog](https://keepachangelog.com)。

## [0.2.0] — 2026-06-30

Main: 在 v0.1 的框架上长出 ANIMA 的第一个**技能**——陪你下国际象棋。这一版的意义是让 ANIMA 从「单步操作世界」升级到「维持一个长程闭环任务」，并立住「大脑通用、任务专属下沉」的分层：通用主循环只管思考与编排，怎么下棋住在技能 / 行为树 / 世界里，绝不上浮到大脑。

Features:
1. 第一个示例世界 sim-chess：一张能托管国际象棋 / 五子棋 / 围棋的「棋桌」，自己握真值、判终局、跑内置棋手；但对大脑**只给画面**，逼大脑靠视觉读盘、经 take_seat 选边就座——和真机摄像头一样的处境。
2. Chess Mode（首个技能）：用「技能（程序性剧本）+ 对弈行为树」维持一整局的闭环，走哪步由引擎定、循环由行为树维持，大脑只在解说和高层判断时被叫出来；换棋种只换一个适配器。
3. 通用运行时与安全：把「人在环路」做成可复用基件（行为树挂起→问人→带答案恢复，且会超时安全中止），并把安全闸升成分级（放行 / 需人批 / 拦截）。
4. 独立 eval 记分台：单独运行、只读对弈档案，按主流象棋标准（ACPL 等）给大脑的棋力打分——可复现、不碰主程序。

Notes: 仍处 1.0 之前，协议可能调整。失败恢复（抓取滑了 / 碰倒了才谈得上的纠错）留待上真机的版本兑现，本版不假装已实现。

## [0.1.0] — 2026-06-27

Main: ANIMA Zero 首个版本。确立「认知与世界分离」的核心架构——ANIMA 作为认知系统只负责思考与决策,World(世界)作为独立实体负责感知与执行,两者通过标准协议 AWI 对接。

Features:
1. 制定 ANIMA 与 World 之间的标准协议——AWI(Anima World Interface);借助 AWI,ANIMA 可以接入不同的世界实体,无论虚拟世界还是真实世界都可以,换世界只需换一个地址,认知端无需改动。
2. 定义「World(世界)」概念:世界是一个独立运行的实体,自己声明能力、负责感知与执行;ANIMA 不触碰其内部,只隔着 AWI 观测与操作它。
3. 设计 ANIMA 的通用认知主循环:先观察世界、再思考、经安全核验后行动、行动后再次观察,如此闭环;操作世界采用大模型原生的工具调用能力,而非在提示词里硬写格式。
4. 建立会话(Session)机制:一次任务对应一个会话、记忆保存在本地,并可在对话中随时切换大脑(支持 5 个视觉大脑,其中一个可在本地离线免费运行)。
5. 搭建认知外围件:行动前的确定性安全闸、对话上下文管理、结构化执行轨迹,以及一个可视化的 AWI 调用监控面板。
6. 实现首个示例世界 sim-desk(虚拟桌面 + 笔 + 画布,提供移动笔、绘制、擦除三种能力),用于端到端验证整套协议。
7. 设计初版 Web 交互界面:三栏布局(会话、实时画面、对话),支持流式输出与执行轨迹的折叠查看。

Notes: 仍处早期阶段,1.0 之前协议可能调整;本版为完全重写,取代更早的 ANIMA O1 原型,不复用其代码。
